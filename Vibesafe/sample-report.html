HIGH Content-Security-Policy (CSP) missing chatflow-ai.vercel.app ▶
Without a CSP header, any injected script (via XSS, a compromised dependency, or a third-party widget) runs with full page access. An attacker can steal session tokens, read chat history, and exfiltrate API keys entered by users.
How to fix
// next.config.js — add to headers()
{
  key: 'Content-Security-Policy',
  value: "default-src 'self'; script-src 'self' 'unsafe-inline'; connect-src 'self' https://api.openai.com;"
}
HIGH Domain @chatflow-ai.com found in 3 data breach(es) admin@chatflow-ai.com ▶
The domain was found in 3 breach databases (Collection #1, LinkedIn, Adobe). Credentials from these breaches are actively used in credential-stuffing attacks. If users reuse passwords, their accounts are at immediate risk.
How to fix
1. Enforce strong password requirements (min 12 chars, complexity).
2. Enable 2FA — even TOTP is sufficient.
3. Force a password reset for all existing accounts.
4. Notify affected users per GDPR/CCPA breach notification obligations.
MEDIUM X-Frame-Options missing chatflow-ai.vercel.app ▶
Your page can be loaded inside an <iframe> on any third-party site. This enables clickjacking — attackers overlay a fake UI on top of your app to trick users into triggering actions (authorising payments, changing email, etc.).
How to fix
// next.config.js
{ key: 'X-Frame-Options', value: 'DENY' }

// or, modern equivalent (superset of X-Frame-Options):
{ key: 'Content-Security-Policy', value: "frame-ancestors 'none';" }
MEDIUM API route /api/chat missing authentication check src/app/api/chat/route.ts ▶
/api/chat does not verify a session token before proxying to OpenAI. Any unauthenticated caller can burn your API quota. On a free-tier key this will exhaust credits in minutes; on a paid key this is a direct financial loss.
How to fix
// src/app/api/chat/route.ts
import { getServerSession } from "next-auth/next";

export async function POST(req: Request) {
  const session = await getServerSession();
  if (!session) {
    return new Response("Unauthorized", { status: 401 });
  }
  // ... rest of handler
}
MEDIUM No rate limiting detected on /api/auth/signin chatflow-ai.vercel.app/api/auth/signin ▶
The sign-in endpoint returns no X-RateLimit-* or Retry-After headers. With no throttle in place, an attacker can attempt thousands of passwords per minute (brute-force / credential-stuffing).
How to fix
// Using @upstash/ratelimit (Vercel Edge)
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, "1 m"), // 10 attempts/min
});

const { success } = await ratelimit.limit(ip);
if (!success) return new Response("Too Many Requests", { status: 429 });
LOW Referrer-Policy missing chatflow-ai.vercel.app ▶
Without a Referrer-Policy, full URLs (including authentication tokens or query params) are sent in the Referer header to every third-party resource your page loads. This can leak session or reset tokens to analytics providers.
How to fix
{ key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' }
LOW X-Content-Type-Options missing chatflow-ai.vercel.app ▶
Without nosniff, browsers may MIME-sniff responses and execute uploaded files as scripts or HTML — particularly relevant if users can upload attachments.
How to fix
{ key: 'X-Content-Type-Options', value: 'nosniff' }
