ICSE Syllabus Content Architecture — The B.L.U.E. Prerequisite Graph

Complete ICSE Class 8–10 Syllabus → Hierarchical Guides → Prerequisite DAG → Personalized Walkthroughs

Content Creation Pipeline

  • • 1. Parse CISCE official syllabus PDF → extract chapter/topic tree
  • • 2. Subject Matter Expert writes guide (Markdown + MathJax)
  • • 3. Prerequisite edges manually tagged + auto-suggested by graph
  • • 4. 3-verifier panel reviews: accuracy, completeness, prereq correctness
  • • 5. Published → Neo4j graph updated → CDN cache invalidated
  • • 6. Video + interactive media attached post-publish

Why Neo4j (Not PostgreSQL) for the Graph

  • • Transitive closure query: "All prerequisites for guide X"
  • • Neo4j: ~2ms for depth-7 DAG with 2,800 edges
  • • PostgreSQL recursive CTE: ~200ms at scale, degrades with depth
  • • Neo4j native: SHORTEST_PATH, variable-length patterns, subgraph matching
  • • Real-time walkthrough generation at 5K req/s peak
  • • Hybrid: PG for guide content + metadata, Neo4j for graph queries only

Verifier System — Content Quality Moat

  • • Per-subject verifier pools (Math Verifiers ≠ Biology Verifiers)
  • • 3-person odd-numbered panels → no ties
  • • Rubric: factual accuracy, syllabus alignment, prerequisite correctness, clarity
  • • Each vote requires written justification (anti-spam/anti-lazy)
  • • Verifiers earn: ₹50-150 per approved review (quality incentive)
  • • Downvote trigger: guide enters re-review (not auto-deleted)

Syllabus Versioning & Update Strategy

  • • CISCE releases syllabus revisions every 2-3 years
  • • Diff engine compares old vs new syllabus → flags affected guides
  • • Guides auto-flagged: "Pending 2027 syllabus review"
  • • Verifier queue prioritized by board exam weightage
  • • Students see: "This guide reflects 2024 syllabus — 2027 update in progress"
  • • Historical versions archived for reference (never deleted)

Assessment Integration

  • • Per-guide practice problems (auto-graded MCQ + manual review)
  • • Chapter-end mock tests (mirrors ICSE board pattern)
  • • Full syllabus mock: 3-hour timed, ICSE marking scheme
  • • Weak-area detection: graph analysis of failed prerequisites
  • • Adaptive: re-route walkthrough to strengthen foundations first
  • • Parent dashboard: progress %, predicted board score, time-to-complete

Offline + Multilingual

  • • PWA: Service Worker caches entire walkthrough + guides
  • • Works offline: read guides, solve problems, track progress
  • • Sync when online: progress + answers uploaded, new content downloaded
  • • Hindi translations: per-guide toggle (English ↔ हिंदी)
  • • Audio guides: TTS for visually impaired / audio learners
  • • Low-data mode: text-only, no video, compressed images
